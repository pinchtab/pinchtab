The scrape fixture site

This site exists so the PinchTab scrape command can be tested against a small but realistic tree of pages. The homepage, the about page, and the guide are ordinary server-rendered HTML: their full text is present in the initial response, so a plain HTTP fetch extracts everything and no browser is required to read them.

One page is different on purpose. The application page ships an empty shell and builds its real content in the browser with JavaScript, the way a single-page app would. A plain HTTP fetch of that page sees almost nothing, so the scraper should route it to a real browser, render it, and recover the content that JavaScript produced.

Because every page links to every other page, the crawler can discover the whole site starting from this homepage without needing a sitemap. The content here is intentionally verbose so that HTTP extraction clears the thin-content threshold and this page stays HTTP-sourced in the report rather than being sent to the browser.
