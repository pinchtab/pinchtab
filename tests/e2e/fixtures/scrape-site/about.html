About this fixture

The scrape fixture site is a deterministic set of pages used by the end-to-end tests for the PinchTab scrape command. It is served by the same nginx container that serves the other fixtures, and its pages never change between runs, which keeps the assertions stable.

This particular page is entirely server-rendered. Everything you can read here is delivered in the first HTTP response, with no client-side rendering involved. That makes it a good control case: the scraper's HTTP extraction should capture the full text, the routing logic should decide that no browser is needed, and the page should appear in the report with its content source recorded as HTTP.

Keeping a few plain pages alongside the JavaScript-only application page lets a single scrape run demonstrate both halves of the pipeline at once: cheap HTTP extraction where it is enough, and browser rendering only where it is actually required.
